"Git Basic Commands in Bash"

"git" is keeping track of the changes in our project it's a version control system used on your local computer.


Frequently used Bash Commands


Where am I ?

pwd: To figure out where you are

Change Directory

cd destination: to change your directory

EX:
cd ~ :to direct you to your Home Directory/Folder
cd E: :to direct you to E partition (Just change the partition name to direct you to any partition)
cd .. :to go back one directory

List files/folders

ls :to list all the folders inside

Start a new Repository

git init :to initialize a git Repository/Project inside your current folder

list all folders including the hidden ones

ls -la :to list all folders including the hidden ones

Clear the screen

clear : to clear everything on the screen

Adding changes to the Repo.

git add . : to add the changes to our repository/Project
Ex :
git add filename.extension : to add a specific file
Adding a file to a repository on a git server goes into 3 stages :

  1. Working copy (in your local computer)
  2. Staging area ( when you use git add , you actually assign the files to be commited to the server)
  3. repository ( when you use git commit , you send the files that in staging area to the repository).

Commiting Changes

NOTE Berfore commiting your first commit you have to configure Git with an Email and User name

git config --global user.email "amr.abdeen@iedu-eg.com"

git config --global user.name "AmrAbdeen"

git commit -m "Write a indication message of changes": to commit our changes so we can keep tracking of it .commiting changes means we keep the first modified version of the project so ever after furthr changes we can get back to it again and modify whatever you want.

Pushing to Github server

git push : to push your local commits to the gitHub server

You can find the repo URL in Quick setup page

git push https://github.com/user/file_name.git

Viewing the Repo. log

git log : to view the commit history

Viewing Users Updates/Changes

git log --author="Name" : to view changes by a specifi user in the project team

Viewing files Status

git status : to view the status of the tracked files ( modified / deleted / new files add / ..etc.)

Exit any Text in the terminal

shift+Q : to exit from any text page

View the difference between file versions

git diff : to show the difference between the original one in the main repository and the modified one on your local computer (which is not commited or added yet ).

stagged files and the original

git diff --staged : to show the difference between the stagged files and the original files in the repository.

File Removal

git rm filename : to remove the file from the repository and the computer itself and you need to commit the removel ;so that command is followed by : git commit -m "removal message"

Rename a file

git mv Oldfilename.extension NewfileName.extension : to rename a file within repository (mv stands for move , think of it as move the name of the old file to the new one "as a metafor to easily remind it").

Moving a file

git mv file.extension folder/ : to move a file into a folder . If you want to move and rename use : git mv Filename.extension folder/newName.extension

Adding (all) modified files to Repo.

git commit -am "message" : to add (all) modified files into the repository directly with no need to put them in the staging area first but if you want to add a specific file to the repository use the regular instruction ( add then commiy)

Restoring a file from the Repo.

git checkout -- file.extension : to replace the file in your local PC with the one in repository ( think of it as restoring the file to its original state)

Unstage

git reset HEAD fileName.extension: to unstage any file in the stagging area

Back to a specific commit

git checkout commitNumber -- file.extension : to get back to a specific commit but firstly, you have to check the git log to know the commitNumber (mostly like a hash number.) and then you have to commit these changes.
